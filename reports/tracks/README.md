# Track References

This file is a manual reference for Codex when choosing one primary track for a paper.
It is not generated by code and it is not a hard taxonomy.

## Reference Buckets

- `memory`: long-context memory, cache, retrieval, persistent state.
- `serving`: online serving, latency, throughput, deployment.
- `runtime`: execution engines, kernels, lowering, runtime systems.
- `scheduling`: batching, queues, admission, orchestration.
- `distributed-inference`: sharding, parallelism, multi-GPU or multi-node inference.
- `edge`: on-device or resource-constrained deployment.
- `compiler`: compilation, graph optimization, code generation.
- `other`: no clear systems bucket.

Use the paper's actual contribution, not keywords alone.
Example: `Titans: Learning to Memorize at Test Time` -> `memory`.
